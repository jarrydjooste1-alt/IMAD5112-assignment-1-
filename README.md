Project Overview

Social Spark is a mobile app I built for my IMAD5112 class. I wrote it in Kotlin using Android Studio. The idea was simple: give people a place to connect, share ideas, and actually have conversations in a community setting.

Building the app checked all the assignment boxes. I had to make sure the app worked well, handled version control with GitHub, and automated builds and tests with GitHub Actions.

Purpose and Features

Purpose:
Social Spark’s all about making social networking straightforward. Share posts, chat with others, and get involved in a community built around shared interests.

Key Features:
- User Authentication: Sign up and log in securely.
- Create & Share Posts: Jot down your thoughts and share them with everyone.
- Interactive Feed: See what others are saying, like their posts, and leave comments.
- User Profile: View your posts and manage your account from one spot.

All of these features make Social Spark a simple and direct way for folks to interact and exchange ideas, minus the clutter.

Design Considerations

I kept a few key things in mind while designing Social Spark:

1. User Experience (UX): I wanted the interface to be really easy to use, even for someone who’s not a “tech person.”
2. Responsiveness: The app adjusts well, whether you’re on a small phone screen or a big tablet.
3. Simplicity: I cut out all the noise—just the essentials like posts, feed, and profile.
4. Performance: I kept things running smooth, quick feed loading, and made sure the app doesn’t chew up battery.

GitHub and GitHub Actions

Everything is version-controlled with GitHub—every bit of code, every change. This kept my project organized and safe.

GitHub Actions:
I set up GitHub Actions to automate builds and deployment. Here’s what it handles:

- Runs automated tests for the main features, like logging in and posting
- Builds APK and AAB files for distribution
- Uploads everything to GitHub so it’s easy to grab and share

Now, every time I push code, the CI system kicks in, tests it, and builds the app for me. It just works.

Challenges and Learnings

This project wasn’t all smooth sailing. A couple of things tripped me up:

1. User Authentication: I wanted it secure and reliable. Researching led me to Firebase Authentication, which turned out to be just what I needed. I managed to get it integrated and working.
2. Gradle Build Issues: Adding new libraries complicated things sometimes. The error logs were confusing at first, but with some Stack Overflow help and lots of version tweaks, I sorted them out.

I learned a lot—mainly about debugging problems, using version control right, and letting GitHub Actions handle my builds.

Future Enhancements

Right now, Social Spark does the basics well. Down the line, I’d like to add:

1. Image & Media Sharing: Let people post photos and videos, not just text.
2. Real-time Messaging: Direct messaging for private conversations.
3. Dark Mode: A nicer look for those who prefer or need a dark interface.
